# Student Management System

A Django-based student management system for keeping student records, viewing a dashboard summary, searching students, and managing student details.

## Features

- User login and logout
- Dashboard with student totals, course counts, academic years, recent students, and course distribution
- Student directory with search
- Add, edit, view, and delete student records
- Django admin support for managing students
- Tests for student search behavior

## Tech Stack

- Python
- Django
- SQLite
- Bootstrap

## Project Structure

```text
student-management-system/
|-- attachment/              # Django project settings and main URLs
|-- students/                # Student app
|   |-- migrations/          # Database migration files
|   |-- static/              # CSS and static assets
|   |-- templates/           # HTML templates
|   |-- admin.py             # Django admin registration
|   |-- apps.py              # App configuration
|   |-- forms.py             # Student form
|   |-- models.py            # Student database model
|   |-- tests.py             # Automated tests
|   |-- urls.py              # App URL routes
|   `-- views.py             # Page logic
|-- manage.py                # Django management command entry point
|-- db.sqlite3               # Local development database
`-- .gitignore               # Files ignored by Git
```

## Setup

1. Install Django if it is not already installed:

```powershell
py -m pip install django
```

2. Apply database migrations:

```powershell
py manage.py migrate
```

3. Create an admin user:

```powershell
py manage.py createsuperuser
```

4. Run the development server:

```powershell
py manage.py runserver
```

5. Open the app:

```text
http://127.0.0.1:8000/login/
```

## Useful Commands

Run tests:

```powershell
py manage.py test
```

Open Django admin:

```text
http://127.0.0.1:8000/admin/
```

## Search

The student search can match:

- Registration number
- First name
- Last name
- Course
- Email
- Phone
- Gender
- Academic year

## Notes

- `db.sqlite3` is a local database file and is ignored by Git.
- `__pycache__/` folders are generated by Python and are ignored by Git.
- The app requires login before accessing the dashboard and student pages.
